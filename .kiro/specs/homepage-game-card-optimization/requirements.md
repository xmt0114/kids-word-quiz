# 需求文档

## 介绍

此功能优化首页的游戏卡片，提供更紧凑和信息丰富的显示。优化包括移除冗余信息、集成教材信息与学习进度可视化，并在提高信息密度的同时保持适合儿童的UI风格。

## 术语表

- **游戏卡片**: 首页显示的交互式卡片组件，代表每个可用的游戏
- **教材信息**: 当前选择的教材相关数据，包括名称、年级和词汇数量
- **学习进度**: 用户的进度数据，包括已掌握词汇、正在学习词汇和剩余词汇
- **进度条**: 显示学习进度分布的三色可视化指示器
- **教材选择器**: 允许用户切换可用教材的可点击界面元素

## 需求

### 需求 1

**用户故事:** 作为用户，我希望看到紧凑的游戏卡片且没有冗余信息，这样我可以专注于重要的游戏详情并拥有更清洁的界面。

#### 验收标准

1. 当首页加载时，系统应当从所有游戏卡片中移除"初级到高级"难度信息
2. 当首页加载时，系统应当从所有游戏卡片中移除"5-15分钟"时长信息
3. 当显示游戏卡片时，系统应当保持与当前实现相同的卡片尺寸
4. 当移除冗余信息时，系统应当保留适合儿童的UI样式和视觉吸引力
5. 当优化卡片布局时，系统应当确保剩余元素的适当间距和对齐

### 需求 2

**用户故事:** 作为用户，我希望在每个游戏卡片上看到我当前的教材名称，这样我可以快速识别每个游戏当前使用的教材。

#### 验收标准

1. 当游戏卡片显示时，系统应当显示该特定游戏的当前教材名称
2. 当显示教材信息时，系统应当仅显示教材名称，不包含年级或其他详细信息
3. 当游戏首次加载时，系统应当自动使用教材列表的第一个作为默认教材
4. 当教材信息加载时，系统应当优雅地处理加载状态而不破坏卡片布局
5. 当教材数据不可用时，系统应当显示适当的后备内容

### 需求 3

**用户故事:** 作为用户，我希望在每个游戏卡片上看到我的学习进度的纯图形化表示，这样我可以快速直观地评估不同游戏和教材的进度。

#### 验收标准

1. 当学习进度数据可用时，系统应当在每个游戏卡片上显示三色进度条
2. 当渲染进度条时，系统应当为已掌握词汇部分使用深绿色
3. 当渲染进度条时，系统应当为正在学习词汇部分使用浅绿色
4. 当渲染进度条时，系统应当为剩余/未解锁词汇部分使用灰色
5. 当显示进度条时，系统应当仅使用图形渲染，不显示任何文字标注或数字

### 需求 4

**用户故事:** 作为用户，我希望能够直接从游戏卡片快速地切换教材，这样我可以快速更改学习材料而无需导航到设置页面。

#### 验收标准

1. 当点击教材名称时，系统应当显示该游戏可用教材的下拉菜单
2. 当从下拉菜单选择不同教材时，系统应当立即更新游戏设置
3. 当教材选择更改时，系统应当刷新新教材的学习进度数据
4. 当下拉菜单打开时，系统应当在点击菜单区域外时关闭菜单
5. 当显示教材选择菜单时，系统应当使用符合儿童UI风格的设计，包括适当的颜色、圆角、阴影和动画效果

### 需求 5

**用户故事:** 作为用户，我希望优化的游戏卡片保持现有的适合儿童的设计语言，这样界面对年轻学习者仍然有吸引力和可访问性。

#### 验收标准

1. 当显示优化卡片时，系统应当保留当前的配色方案和渐变
2. 当显示教材信息时，系统应当使用与现有设计系统一致的字体和样式
3. 当渲染进度条时，系统应当使用与当前UI风格一致的圆角和平滑过渡
4. 当显示交互元素时，系统应当保持与现有设计匹配的悬停效果和动画
5. 当布局卡片内容时，系统应当确保适合儿童用户的适当视觉层次和可读性
